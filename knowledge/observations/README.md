# Knowledge Observations

Observation files are learning candidates created after QC review. They are not
loaded directly during extraction.

Use this folder for deal-specific lessons such as:

- engine result was missing or wrong
- human-verified source cell and value
- proposed reusable lesson
- evidence source and review status

Recurring accepted observations are distilled into `knowledge/patterns/*.json`.
Only promoted, low-contradiction patterns should affect future runtime behavior.
